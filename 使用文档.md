# OpenClaw 控制台使用文档

这份文档对应当前桌面版界面，按现在的软件真实结构来写：

- `首页`
- `AI 服务中心`
- `聊天`
- `调教中心`
- `修复中心`

目标不是讲底层概念，而是让第一次接触 OpenClaw 的用户也能按步骤跑通。

## 一、软件是干什么的

这个软件是一个面向小白的 `OpenClaw 图形化控制台`，主要解决 4 件事：

1. 帮你检查环境并安装 OpenClaw。
2. 帮你接入 AI 服务，不用手写配置文件。
3. 帮你创建 Agent、配置渠道、绑定网关。
4. 帮你在本地聊天，或把 Telegram、QQ、飞书等渠道接到对应 Agent。

简单理解：

- `Agent`：你的不同助手身份，比如 `main`、`code-agent`
- `渠道`：Telegram、QQ、飞书、Discord、钉钉这些消息入口
- `网关`：把渠道和 Agent 真正接起来的运行入口

## 二、第一次使用怎么跑通

建议第一次使用严格按这个顺序：

1. 打开 `首页`
2. 完成环境检测和安装
3. 进入 `AI 服务中心` 填好 Key
4. 返回 `首页` 或直接进入 `聊天`
5. 先用本地聊天确认 AI 可正常回复
6. 再去 `调教中心 -> Agent 管理`
7. 最后去 `调教中心 -> 渠道配置` 配 Telegram、QQ、飞书等外部渠道

如果只想先跑通，不碰外部渠道，做到第 5 步就够了。

## 三、页面总览

### 1. 首页

首页是总入口，主要看 3 件事：

- 当前环境是否可用
- AI 是否已经接入
- 当前是否已经能开始聊天

首页适合做这些动作：

- 看环境检测结果
- 补装 OpenClaw
- 快速跳转到 `AI 服务中心`
- 快速跳转到 `聊天`
- 出问题时直接进入 `修复中心`

如果你是第一次用，先看左侧的 `首次跑通路径`。

### 2. AI 服务中心

这是你软件商业化最重要的入口，也是普通用户最常用的页面。

当前推荐流程：

1. 选择服务渠道
2. 选择模型方案
3. 填入 API Key
4. 点击 `验证密钥`
5. 点击 `保存并启用服务`
6. 点击 `立即开始试聊`

目前页面默认是小白模式，只保留必要动作。  
如果你只是想先用起来，不要先碰高级选项。

### 3. 聊天

聊天页是当前真正的对话主界面：

- 左侧是 Agent 列表
- 右侧是当前 Agent 的会话内容
- 顶部可以切换路由模式、执行方式、网关

适合做这些事：

- 直接和某个 Agent 聊天
- 切换不同 Agent 查看各自对话
- 测试当前 AI 配置是否生效
- 测试某个 Agent 的风格、能力、记忆是否正常

建议第一次先用聊天页确认：

- 能发出消息
- 能正常收到回复
- 切换 Agent 不会报错

### 4. 调教中心

调教中心是日常配置区，现在按小白使用习惯拆成几块：

- `Agent 管理`
- `渠道配置`
- `Skills`
- `记忆`
- `模板`
- `高级设置`

建议理解成：

- `Agent 管理` 负责管助手本身
- `渠道配置` 负责把 Telegram、QQ、飞书这些入口接进来
- `Skills` 负责能力扩展
- `记忆` 负责长期信息
- `高级设置` 负责模型策略和维护工具

### 5. 修复中心

修复中心是排错页面。  
当你遇到安装失败、网关异常、技能修复失败、渠道不通时，优先来这里。

适合做这些事：

- 一键体检
- 一键修复
- 查看运行状态
- 导出诊断信息

如果不确定问题出在哪，先跑一次体检再说。

## 四、最推荐的新手使用流程

### 路线 A：只想先本地聊起来

1. `首页` 完成环境检测
2. 安装 OpenClaw
3. 进入 `AI 服务中心`
4. 选一个服务渠道
5. 填 Key 并验证
6. 点击 `保存并启用服务`
7. 进入 `聊天`
8. 给默认 Agent 发一条消息

这条路线最快，适合第一次确认软件本身没问题。

### 路线 B：想做多 Agent

1. 先完成路线 A
2. 进入 `调教中心 -> Agent 管理`
3. 新建 Agent
4. 给 Agent 改名
5. 设定默认 Agent
6. 回到 `聊天` 测试不同 Agent

建议先把 Agent 名称改成一眼能懂的名字，比如：

- `通用助手`
- `代码助手`
- `写作助手`

### 路线 C：想接入外部渠道

1. 先完成路线 A
2. 如果需要多个身份，先完成路线 B
3. 进入 `调教中心 -> 渠道配置`
4. 选择目标渠道
5. 按 Agent 填对应凭据
6. 点击底部固定操作条中的 `保存并应用`
7. 点击 `测试连通`
8. 点击 `启动当前 Agent 网关`
9. 回到对应外部渠道测试消息

## 五、AI 服务中心怎么用

### 1. 服务渠道选择

当前页面会优先展示适合小白直接开用的接入方式。  
一般按这个原则选：

- 想便宜稳定，优先默认推荐渠道
- 想长文本能力更稳，选 Kimi
- 想后续接入你自己的中转线路，后面可以从这里扩展

### 2. 模型方案选择

页面展示的是当前产品预设模型。  
小白用户建议先不要纠结参数，直接选推荐项。

### 3. API Key

常用动作只有 3 个：

- `粘贴`
- `验证密钥`
- `保存并启用服务`

推荐顺序：

1. 粘贴 Key
2. 先验证
3. 验证通过后保存

### 4. 什么时候算配置成功

出现以下任一情况，基本就算成功：

- 页面显示 `已接入，可直接开始聊天`
- 能点击 `立即开始试聊`
- 进入 `聊天` 后能正常收到模型回复

## 六、聊天页怎么用

### 1. 左侧 Agent 列表

左侧是不同 Agent 的会话入口。  
点击谁，就和谁聊天。

每个 Agent 的聊天记录是分开的。  
如果你创建了多个 Agent，不同 Agent 看到的会话内容本来就不同。

### 2. 顶部常用设置

顶部一般会看到这些：

- `路由模式`
- `执行方式`
- `网关`

如果你不确定怎么选，建议这样：

- `路由模式`：先用默认
- `执行方式`：先用默认推荐项
- `网关`：保持当前 Agent 自动选中的可用网关

### 3. 聊天页适合做什么

- 验证 AI 服务有没有接成功
- 验证某个 Agent 有没有问题
- 看未读和最近会话
- 验证网关在线时的真实对话表现

### 4. 聊天页卡顿时怎么判断

先看是否处于这些情况：

- 正在持续生成回复
- 当前 Agent 历史消息过多
- 网关正在同步历史
- 浏览器本身资源占用较高

如果只有回复过程中卡，回复结束就恢复，通常属于消息流式渲染带来的短时压力，不一定是配置错误。

## 七、调教中心怎么用

### 1. Agent 管理

这是当前的小白默认页，主要只保留 5 个核心动作：

- 查看 Agent 列表
- 改名
- 设默认
- 新建
- 删除

推荐做法：

1. 先保留 `main`
2. 新建 1 到 2 个真正有用的 Agent
3. 用清晰名称区分用途
4. 不要一上来建太多

### 2. 渠道配置

这是把外部消息入口接到 Agent 的页面。

当前页面结构可以这样理解：

- 左侧：渠道标签
- 右侧：当前渠道的配置内容
- 底部固定条：最常用动作

#### 左侧渠道状态怎么看

每个渠道会显示一种状态：

- `待补全`：还没填好凭据
- `已配置`：凭据填好了，但还没进网关
- `已连通`：已经进入当前 Agent 网关绑定

#### 右侧配置区怎么用

右侧按渠道填写对应凭据。  
你不需要先理解底层 JSON，只要按界面填即可。

对于大多数用户，流程固定是：

1. 选择渠道
2. 按 Agent 自动生成配置项
3. 给每个 Agent 填凭据
4. 看底部固定条
5. 点 `保存并应用`
6. 点 `测试连通`
7. 点 `启动当前 Agent 网关`

#### 底部固定操作条是做什么的

底部固定条现在只保留 3 个动作：

- `保存并应用`
- `测试连通`
- `启动当前 Agent 网关`

同时会显示：

- 当前 Agent
- 当前渠道
- 当前网关
- 最近一次操作结果

这意味着你不用再到处找按钮，也不用操作后再往上翻结果。

### 3. 首次配对审批

像 Telegram、飞书、QQ 这类渠道，第一次可能需要配对或审批。

如果用户在外部渠道看到配对码：

1. 回到 `调教中心 -> 渠道配置`
2. 切到对应渠道
3. 在 `首次配对审批` 区域输入或查询配对码
4. 点击批准

审批完成后，再回外部渠道发消息测试。

### 4. Skills

Skills 是 Agent 的扩展能力。  
如果你当前只是想把软件跑通，可以先不动。

建议使用顺序：

1. 先确认聊天正常
2. 再安装 skills
3. 安装后做一次修复或检测
4. 如果状态异常，再去修复中心

### 5. 记忆

记忆用于保存长期信息。  
如果你希望 Agent 记住固定偏好、背景、规则，可以在这里配置。

### 6. 高级设置

高级设置已经拆成 2 类：

- `模型策略`
- `维护工具`

如果你不是进阶用户，建议默认不要展开。  
先把 `Agent 管理` 和 `渠道配置` 跑通，再回来碰这里。

## 八、常见渠道怎么配置

### 1. Telegram

推荐流程：

1. 进入 `调教中心 -> 渠道配置`
2. 选择 `Telegram`
3. 按 Agent 自动生成
4. 给每个 Agent 填 `Bot Token`
5. 需要的话填 `chatId`
6. 点击底部 `保存并应用`
7. 点击底部 `测试连通`
8. 点击底部 `启动当前 Agent 网关`
9. 去 Telegram 给机器人发消息

如果有多个 Telegram Bot，不要让多个网关抢同一个 Token。

### 2. QQ

当前 QQ 配置使用的是：

- `AppID`
- `AppSecret`

界面会自动按 OpenClaw 需要的格式处理，不需要你自己手动拼接命令。

推荐流程：

1. 选择 `QQ`
2. 按 Agent 自动生成
3. 填 `AppID`
4. 填 `AppSecret`
5. 保存并应用
6. 测试连通
7. 启动当前 Agent 网关
8. 如提示配对码，再到审批区处理

### 3. 飞书

飞书一般最容易卡在回调和首次配对。

建议顺序：

1. 填好应用凭据
2. 确认回调地址配置正确
3. 在页面内审批首次配对码
4. 保存并应用
5. 启动网关
6. 到飞书侧测试

如果提示 `access not configured`，通常不是模型问题，而是还没完成首次配对或权限放行。

## 九、修复中心怎么用

修复中心适合这几类问题：

- 环境缺 Node / Git / OpenClaw
- OpenClaw 已安装但命令不可用
- 网关启动失败
- 渠道配置了但不能对话
- Skills 状态异常

建议排错顺序：

1. 先点 `一键体检`
2. 看结果里先报哪一项
3. 能自动修就先自动修
4. 修完再测一次
5. 还不行再手动看网关和渠道

## 十、常见问题

### 1. 打开软件后，还是提示环境不完整

先确认：

- Node.js 已安装
- Git 已安装
- 安装后重新打开软件

如果命令已经能用但软件仍未识别，去 `修复中心` 重新体检一次。

### 2. 保存 AI 配置后还是不能聊天

优先检查：

1. Key 是否验证通过
2. 当前模型是否已选中
3. 是否已经点击 `保存并启用服务`
4. 聊天页是否能看到当前 Agent

### 3. 聊天页能打开，但没回复

先区分两种情况：

- 本地聊天都不回复：优先检查 AI 服务配置
- 本地聊天能回复，外部渠道不回复：优先检查渠道配置和网关

### 4. 外部渠道不回复，但聊天页正常

通常排查顺序是：

1. 渠道凭据是否填对
2. 是否点过 `保存并应用`
3. 是否做过 `测试连通`
4. 当前 Agent 网关是否已启动
5. 是否存在首次配对未审批

### 5. 提示配对码或 access not configured

这类问题不要去 AI 页面改参数。  
直接去 `调教中心 -> 渠道配置` 的对应渠道里处理首次配对审批。

### 6. 浏览器打开对话页失败或拒绝访问

先试：

1. 确认网关是否已启动
2. 等几秒再打开
3. 从聊天页或首页重新触发打开
4. 检查是否被本机安全软件拦截

### 7. Telegram 提示冲突或 getUpdates 异常

大多数情况是多个网关或多个实例在抢同一个 Bot Token。  
原则上一个 Telegram Token 只应由一个有效网关轮询。

### 8. 不知道自定义配置路径该填什么

这里填的是 `OpenClaw 配置目录`，不是程序安装目录。  
目录里应该能找到 `openclaw.json` 之类的状态文件。

如果你不确定，优先：

1. 留空使用默认路径
2. 或使用自动检测

### 9. Skills 修复后状态还是不对

先看它显示的是：

- 已修复
- 已尝试修复
- 仍缺依赖

如果是 `仍缺依赖（已尝试修复）`，说明程序已经尝试过自动修，但依赖问题还没有完全解决，这时应继续看具体报错。

## 十一、推荐给新手的默认玩法

### 玩法 1：单 Agent 本地聊天

适合第一次用软件的人。  
只做：

- AI 服务中心配置好 Key
- 用默认 `main` Agent 聊天

### 玩法 2：两个 Agent 分工

适合已经跑通本地聊天的人。  
例如：

- `main`：通用聊天
- `code-agent`：代码和技术问答

### 玩法 3：单 Agent 多渠道

适合想把一个助手接入多个入口的人。  
例如同一个 Agent 同时接：

- Telegram
- QQ
- 飞书

这也是当前软件最推荐的小白外部接入玩法。

## 十二、命令行和脚本用户说明

如果你不用桌面界面，也可以使用附带脚本或命令行。

常见命令：

```bash
openclaw gateway start
openclaw gateway stop
openclaw gateway status
openclaw doctor
openclaw onboard
```

这部分更适合愿意手动排错的用户。  
普通用户优先使用图形界面即可。

## 十三、建议

第一次使用时，最推荐记住这三条：

1. 先本地聊天跑通，再去接外部渠道。
2. 先把 Agent 名称和用途理清，再去做多渠道接入。
3. 出问题先去 `修复中心`，不要一上来手改底层配置。

如果你后续还要把文档继续做完整，下一步最值得补的是：

- 每个渠道各写一份独立配置教程
- 单独写一份 `排错手册`
- 单独写一份 `Docker 部署文档`
